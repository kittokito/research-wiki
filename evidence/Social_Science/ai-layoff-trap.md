# The AI Layoff Trap からの抽出

## 主要な主張
- 競争的環境において需要の外部性が合理的企業を自動化軍拡競争に閉じ込める（automation arms race） [source](../../sources/Social_Science/ai-layoff-trap.md)
- 各企業は自動化のインセンティブを持つが、全体としては集団的最適を大幅に超える労働者代替が発生する（囚人のジレンマ構造） [source](../../sources/Social_Science/ai-layoff-trap.md)
- 過剰自動化の損失は労働者だけでなく企業オーナーにも及ぶ [source](../../sources/Social_Science/ai-layoff-trap.md)
- 競争が激化し「より優れた」AIが登場するほど過剰自動化は増幅される [source](../../sources/Social_Science/ai-layoff-trap.md)

## 検討された政策手段と評価
| 政策手段 | 過剰自動化の解消 | 備考 |
|---|---|---|
| 賃金調整 | 不可 | 需要外部性を解消しない |
| 自由市場参入 | 不可 | 競争激化がむしろ問題を増幅 |
| 資本所得税 | 不可 | 外部性の根本に対処しない |
| 労働者持株（equity participation） | 不可 | インセンティブ構造を変えられない |
| ユニバーサル・ベーシック・インカム（UBI） | 不可 | 需要は補填しうるが自動化インセンティブは残る |
| リスキリング（upskilling） | 不可 | 代替可能タスクの問題は残る |
| コース的交渉（Coasian bargaining） | 不可 | 取引費用や参加者数の問題 |
| **ピグー税（Pigouvian automation tax）** | **可能** | 外部性を内部化する唯一の手段 |

## 主要な貢献
- AI労働代替の「知っていても止められない」メカニズムをタスクベース競争モデルで形式化
- 7つの主要政策提案を統一的フレームワークで評価・棄却し、ピグー税の必要性を理論的に導出
- 「AIの結果への対処」だけでなく「自動化を駆動する競争インセンティブそのもの」への政策対応の必要性を提起

## 制限・注意点
- 理論モデルに基づく分析であり、実証データによる検証は行われていない
- タスクベースモデルの仮定（タスクの代替可能性、需要弾力性等）に結果が依存
- ピグー税の最適税率の算定方法や政治的実現可能性は未検討
- AI導入による新タスク・新産業の創出効果（compensating effect）のモデル化が限定的な可能性
